# 群聊消息组与顶部讨论精简设计

## 目标

群聊消息按连续发送者形成稳定的视觉组。头像、昵称和内容属于同一个布局容器；图片与后续文字沿同一内容左边缘排列。同时进一步精简顶部讨论区域：不显示数量标题，也不展示已结束项目。

## 消息分组

- 以消息列表当前顺序为准，相邻且发送者身份相同的消息合并为一个消息组。
- 发送者优先以 `senderMemberId` 判定；缺少成员 ID 时使用 `senderName`，并保留 `mine` 方向判断。
- 每个消息组只渲染一次头像和昵称。
- 头像位于左侧固定列；昵称与全部消息内容位于右侧内容列。
- 昵称是内容列的 header，与图片、文件和文字的左边缘严格对齐。
- 同组中的图片、文件、文字、评评理生命周期消息按原时间顺序纵向排列。
- 每条消息仍保留独立的长按菜单、选择、删除和定位属性，不改变现有交互能力。

## 视觉规格

- 昵称字号 `12px`，字重 `500`，颜色 `rgba(0, 0, 0, 0.45)`。
- 昵称与第一条内容间距 `5px`。
- 同组内容保持紧凑间距，不增加阴影、描边或额外卡片背景。
- 图片继续使用现有尺寸、圆角和预览行为，仅改变其在消息组中的对齐关系。
- 本次不调整输入框、群聊顶部栏或成员头像栏。

## 顶部讨论区域

- 删除紧凑状态下的“置顶 · N”标题。
- 只显示仍在进行中的投票和评评理。
- 已结束的投票和评评理不再出现在顶部紧凑区域，但完整记录继续保留在聊天时间线中。
- 进行中的项目继续沿用现有数量、标题、箭头、最多三条和点击定位能力。

## 滚动与兼容性

- 消息分组只改变消息列表内部 DOM 结构，不改变 `.chat-fullscreen-messages` 的滚动容器、尺寸或键盘底部间距。
- 保持当前左右消息方向语义；当前本人消息的视觉方向不在本次范围内调整。
- 空消息列表、单条消息、发送者交替、连续图片及图片后接文字均应正常渲染。

## 验证

- 静态回归覆盖分组生成、每组单头像/单昵称、昵称样式和顶部已结束项目过滤。
- TypeScript 检查及群聊回归脚本通过。
- 在移动端视口验证连续图片和图片后接文字的对齐、长按交互以及消息区独立滚动。
- 验证公网部署返回正常且无相关控制台错误。
