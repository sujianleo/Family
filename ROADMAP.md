# Fanmili Roadmap

Fanmili is becoming a durable, private, and self-hosted space for family life. This roadmap is organized by capability maturity rather than fixed release dates. Real family feedback, data safety, and maintainability take priority over feature count.

## Now · Family Workspace Foundation

- [x] Single-container SQLite deployment
- [x] Family members and local accounts
- [x] Personal tasks, owners, reminders, and completion states
- [x] Photo, PDF, and family document storage
- [x] Family group chat and multi-member ownership
- [x] Mobile-first PWA with monochrome and dopamine themes
- [x] Optional DeepSeek API with confirmation before important actions

## Next · Long-term Family Memory

- [ ] Structured memory for family events, preferences, and relationships
- [ ] Automatic classification, tagging, and duplicate detection for photos and documents
- [ ] Source-grounded semantic search and RAG
- [ ] Review, edit, expiration, and deletion controls for memory candidates
- [ ] Family member profiles and child growth records

Completion criteria: AI answers can be traced back to original family records, and family members decide what is worth keeping as long-term memory.

## Later · Everyday Connections

- [ ] Family calendar and recurring plans
- [ ] Voice input, spoken reminders, and family briefings
- [ ] Local language models and custom model providers
- [ ] Encrypted backup, recovery, and device migration
- [ ] Multi-device synchronization and offline conflict handling

Completion criteria: external integrations preserve family boundaries, explicit authorization, and a recoverable data trail.

## Future · Family AI OS

- [ ] Native iOS and Android experiences
- [ ] Smart-home events and device state
- [ ] Multiple family spaces with fine-grained permissions
- [ ] Auditable family Agent automation
- [ ] An open integration and plugin ecosystem

Completion criteria: automation is previewable, reversible, and traceable by default. AI never acts on behalf of a family member without clear authorization.

## How Priorities Are Chosen

1. Does it solve a recurring problem for a real family?
2. Can it preserve self-hosting and data ownership?
3. Does it have clear confirmation, reversal, and audit boundaries?
4. Can completion be proven through tests and the real interface?

Items that are ready for implementation will become GitHub Issues. Larger efforts that span multiple issues will be tracked through GitHub Projects.
